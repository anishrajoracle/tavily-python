# Modifications: OracleDB and MongoDB Support in `TavilyHybridClient`

## Goal

Make `TavilyHybridClient` support two first-class local database providers:

```python
db_provider="oracle"
db_provider="mongodb"
```

The change is scoped to loosening the provider coupling in `tavily/hybrid_rag/hybrid_rag.py`. It does not introduce a separate package, wrapper, or cache product. Both providers live in the main hybrid RAG client.

## Design Principle

OracleDB and MongoDB are treated as peer database providers, with OracleDB listed first in the public examples and provider branches.

The code now uses explicit provider branches:

```python
if self.db_provider == "oracle":
    ...
elif self.db_provider == "mongodb":
    ...
else:
    raise ValueError(...)
```

This avoids making either provider look like the default and the other one look like an exception.

## What Changed

### Constructor

The constructor now accepts:

```python
db_provider: Literal["oracle", "mongodb"]
```

OracleDB arguments:

```python
TavilyHybridClient(
    api_key="...",
    db_provider="oracle",
    connection=oracle_connection,
    table_name="TAVILY_DOCUMENTS",
    embeddings_field="EMBEDDINGS",
    content_field="CONTENT",
)
```

MongoDB arguments:

```python
TavilyHybridClient(
    api_key="...",
    db_provider="mongodb",
    collection=mongo_collection,
    index="vector_search",
    embeddings_field="embeddings",
    content_field="content",
)
```

### Local Search

OracleDB local search uses Oracle Database 23ai vector search:

```sql
VECTOR_DISTANCE(EMBEDDINGS, :query_vector, COSINE)
```

The Oracle-specific search implementation is isolated in:

```python
_search_oracle(...)
```

MongoDB local search continues to use the existing Atlas Vector Search aggregation:

```python
collection.aggregate([
    {"$vectorSearch": ...},
    {"$project": ...},
])
```

### Saving Foreign Results

OracleDB saving uses:

```python
_insert_oracle_documents(...)
```

MongoDB saving continues to use:

```python
self.collection.insert_many(documents)
```

Oracle vector values are converted to:

```python
array.array("f", values)
```

for `python-oracledb` vector binding.

## Identifier Safety for OracleDB

Oracle table and column names cannot be passed as ordinary bind variables. To avoid unsafe SQL interpolation, Oracle identifiers are validated before they are used.

Allowed identifier pattern:

```text
^[A-Za-z_][A-Za-z0-9_]*$
```

This validation applies to:

- `table_name`
- `embeddings_field`
- `content_field`
- document keys used during Oracle inserts

## Dependencies

`setup.py` now exposes optional extras for both database providers:

```python
extras_require={
    "oracle": ["oracledb"],
    "mongodb": ["pymongo"],
}
```

The base Tavily SDK does not force-install either database driver.

Install OracleDB support:

```bash
pip install -e ".[oracle]"
```

Install MongoDB support:

```bash
pip install -e ".[mongodb]"
```

## Example

Added:

```text
examples/hybrid_rag_oracle.py
```

This mirrors the existing MongoDB example shape, but uses a `python-oracledb` connection and `db_provider="oracle"`.

## Tests

Added:

```text
tests/test_hybrid_rag_oracle.py
```

The tests use a fake Oracle connection/cursor, so they verify the OracleDB path without requiring a live database.

They check:

- OracleDB search emits `VECTOR_DISTANCE`.
- Oracle vector binds use `array.array("f", ...)`.
- Oracle inserts convert embeddings correctly.
- Unsafe Oracle identifiers are rejected.

## Expected Oracle Table Shape

```sql
CREATE TABLE tavily_documents (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    content CLOB,
    embeddings VECTOR(*, FLOAT32),
    site_url VARCHAR2(1000),
    site_title VARCHAR2(500),
    added_at TIMESTAMP DEFAULT SYSTIMESTAMP
);
```

Required columns:

```text
CONTENT
EMBEDDINGS
```

Any additional keys returned by a custom `save_foreign` function must correspond to real Oracle table columns.

## Non-Goals

This change does not create a wrapper package or a separate Oracle-only product. The point is to support OracleDB and MongoDB inside the existing hybrid RAG client as two first-class database provider choices.
